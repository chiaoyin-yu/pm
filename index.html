我是具備心理學、市場研究與網頁專案管理背景的 Project Manager。大學期間修習「消費者心理學」，讓我習慣從使用者行為、認知負荷與決策焦慮出發理解專案需求——而不只是從功能清單出發。

在市場研究公司，我學會了如何把真實的使用者語言翻譯成對業務有意義的洞察；在網頁專案管理的工作中，我負責與客戶討論需求、將需求轉譯給工程團隊，並追蹤到專案上線。

這段跨域經歷讓我成為一個真正能銜接研究與執行的 Project Manager——知道怎麼問對問題，也知道怎麼把答案推進到交付。

「最好的專案決策，來自對人的深刻理解，加上對執行現實的清醒認識。」

目前我正在尋找 Project Manager 的機會，期待在一個重視使用者思維、注重跨部門協作的團隊中繼續成長。

台灣電力公司區營業處網站重建
——從標案到全台分處官網上線

這是一個從政府標案期間就開始深度介入的專案。台電各區分處原有的舊版網站系統老舊，需要全面遷移至新版共構平台。我從需求釐清、內容轉移、頁面設計規劃，到與客戶反覆確認後台流程與操作需求，全程推進直到正式上線。

查看上線成果 → branch.taipower.com.tw
專案角色：標案銜接 · 需求協調 · 遷移執行
  • 從標案期間即介入，提前理解政府客戶的需求框架與法規限制（政府網站營運規範、無障礙標章）
  • 主導舊版網站內容盤點與轉移計畫，確保資料完整性與新版架構的對應邏輯
  • 協調客戶端各區分處的頁面設計需求，整合多版型差異並統一後台管理邏輯
  • 反覆與客戶對談，確認後台操作流程符合各分處人員的實際使用情境
  • 管理時程與交付節點，確保專案在標案期限內完成上線並通過驗收
專案挑戰與解法
  • 多區分處同步推進、需求各異：建立統一的需求收集與確認機制，避免版本混亂
  • 政府無障礙規範嚴格：全程確認頁面符合無障礙 A 等級標準，最終取得標章認證
  • 後台需求複雜（選單自訂、多版型、共構平台架構）：拆解需求優先順序，與工程師確認每個模組的開發邊界
  • 客戶更新習慣差異大：設計符合非技術人員操作的後台流程，降低上線後的維護成本

Project Management Stakeholder Mgmt Information Architecture
Project Impact 全球官網成功上線，導覽邏輯從品牌內部分類改為使用者搜尋路徑，視覺符合高端品牌定位。

MAPEX 全球官網重整
——從需求訪談到專案上線

MAPEX 是 KHS 集團旗下的全球鼓具品牌。原有網站架構老舊，無法支援品牌的高端定位與複雜產品線。這個專案的挑戰在於：客戶對「想要什麼」的描述非常模糊——我的工作是把它轉化成可以執行的專案規格。

查看上線成果 → mapexdrums.com
專案角色：需求翻譯 · 架構設計 · 跨團隊協調
  • 主導需求訪談，將客戶描述的「更時尚、更直覺的產品路徑」拆解成具體的功能規格與頁面架構
  • 以「使用者如何找到對的鼓組」重新設計導覽邏輯，而非依品牌內部的 SKU 編號分類
  • 簡化導覽層級，降低使用者在多產品線之間的認知負荷（Cognitive Load）
  • 協調客戶端、設計師與後端工程師，確認模組選型、頁面流程與開發優先順序
  • 全程管理交付物與時程，確保需求在傳遞過程中不失真
專案決策亮點
  • 打破客戶原有「依產品系列分頁」的慣性，改為依使用情境（初學 / 進階 / 專業）切入，大幅降低選擇障礙
  • 建立可擴展的內容結構，支援後續多地區、多語系的內容增加，避免後期大改成本
  • 視覺方向對齊品牌高端定位，強化第一印象的信任感——這是可以量化的 UX 目標，不只是美感偏好
03
Research Project Consumer Research Stakeholder Briefing
Project Impact 釐清三個不同使用者族群的決策模式差異，直接影響客戶的產品定位、行銷溝通與 SKU 規劃方向。

烹飪家電消費者行為研究
——化解選擇過剩的決策焦慮

客戶面臨的核心問題是：產品線越做越豐富，轉換率卻沒有相應提升。使用者在選擇時出現明顯的決策焦慮，但不確定痛點在哪一個環節。我設計了三階段研究，找出問題根源，並為後續策略提供具體方向。

專案角色：問題定義 → 研究設計 → 成果簡報
  • 與業務端對齊核心問題：「為什麼功能更多，轉換率沒有提升？」
  • 設計三階段研究流程（分組招募 → 訪前 Homework → 深度訪談），確保資料反映真實行為而非臨場建構
  • 依年齡、有無小孩、下廚頻率、購買意願分層招募，確保樣本能捕捉族群差異
  • 將訪談洞察轉化為策略建議：每個族群需要不同的資訊結構與信任建立方式
Key Findings → 策略建議
  • 選項過多是根本問題，使用者需要清楚的比較框架，而非更多功能列表
  • 信任感的建立優先於功能數量——口碑、品牌熟悉度與資訊一致性才是關鍵轉換因子
  • 「有孩子 / 下廚頻率高」族群與「單身 / 下廚少」族群的決策邏輯完全不同，單一溝通策略無效
  • 建議方向：以使用情境重組導覽，取代以功能規格為主的選擇頁面
